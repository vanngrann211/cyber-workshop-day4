Concept

What Is a Network Share?

A network share is a folder one computer makes available to other computers on the same network. Your instructor's laptop is sharing a folder right now, and you're about to browse it from your own machine.

THE KEY IDEA

Once you connect, that remote folder acts almost like a folder on your own computer. Everything inside it is reachable. That convenience is also a security risk, which is the whole point of today.

Concept

Hidden Does Not Mean Safe

Some flags today are marked hidden. A hidden file just doesn't show up in normal browsing. But one command flag reveals every hidden file instantly.

REMEMBER THIS FOR THE DEBRIEF

Hiding a file is like putting a "do not look" sticker on a box. It stops a casual glance. It stops nobody who is actually looking. Keep that in mind as you hunt.

Setup

Step 2: Open CMD and Map the Drive

Press Win+R, type cmd, press Enter. Then connect to the share:

ping 192.168.137.1 net use Z: \\192.168.137.1\challenge Z:
WHAT THIS DOES

ping tests that you can reach the instructor's laptop. net use maps their shared folder as your Z drive. Typing Z: then Enter moves you onto that drive. Now you can hunt. Replace the IP with the real one from the board.

Toolkit

Your Four Key Commands

dir /s /a list every file, including hidden, in all folders type filename read what is inside a file findstr /s /i "FLAG" *.* search inside all files for a word attrib filename show if a file is Hidden (H)
THE BIG TWO

dir /s /a is your map: it reveals everything, even hidden files. findstr is your metal detector: it digs inside files for the word FLAG. Master these two and the flags fall.

Flag 1 of 5

Warmup: The Welcome Mat

DIFFICULTY: EASYSKILL: dir /s + type
MISSION

Somewhere on the share is a plain welcome file holding your first flag. Nothing is hidden. Just list the files across all folders and read the obvious one.

HINT 1

Use dir with the /s flag to list files in every subfolder at once.

HINT 2

Look for a file named like a greeting in a public folder, then read it with type.

Flag 2 of 5

The Invisible File

DIFFICULTY: EASYSKILL: the /a flag
MISSION

This flag sits in a file that does NOT appear in a normal dir. It is hidden. You'll need to tell dir to show hidden files too.

HINT 1

A normal dir hides it. Add the /a flag to show ALL files including hidden ones.

HINT 2

Check folders named like documents and old. The hidden file lives deep in a user folder. Then type it.

Flag 3 of 5

Wolf in Sheep's Clothing

DIFFICULTY: MEDIUMSKILL: filter by extension
MISSION

This flag is in a file disguised to look boring and harmless, with an extension like .tmp, as if it were a leftover system file. It is also hidden. Sniff out the imposter.

HINT 1

List everything in bare format with dir /s /a /b, then scan for an odd .tmp file.

HINT 2

Pipe the list into findstr to filter: dir Z:\ /s /a /b then add a pipe to findstr .tmp, then type that file.

Flag 4 of 5

Needle in the Haystack

DIFFICULTY: MEDIUMSKILL: findstr content search
MISSION

This flag is buried inside a file stuffed with junk text. Reading the file by hand is painful. Instead, search INSIDE the files for the word FLAG.

HINT 1

findstr searches inside files, like grep did in Linux. You want it to look in every file.

HINT 2

Use /s to search subfolders and /i to ignore capitalization: findstr /s /i "FLAG" Z:\*.*

Flag 5 of 5

Boss Level: The Vault

DIFFICULTY: HARDSKILL: combine everything
MISSION

The final flag is hidden, renamed to look like a boring backup file, and buried several folders deep. No single trick finds it. You must combine your tools: reveal hidden files, enumerate everything, and spot the one file that does not belong.

HINT 1

Start with dir Z:\ /s /a /b to dump every file path including hidden ones. Read the list carefully.

HINT 2

Look in the deepest nested folders for a file with a boring name like .bak or config. Then type it, or run findstr /s /i "FLAG" Z:\*.* to confirm.

Watch Out

Beware the Decoys

The share is salted with fake files designed to waste your time. Real pen testers face this constantly: not every interesting-looking file is the prize.

  • A readme file with fake login credentials
  • A notes file containing FAKE_FLAG, not a real flag
  • A system log that looks important but holds nothing
THE TELL

Real flags follow the exact format FLAG and curly braces. Anything saying FAKE_FLAG or tryharder is bait. If the submit box rejects it, you may have grabbed a decoy.

Debrief 1

Hiding Is Not Securing

Every flag today was hidden, renamed, or buried. None of that stopped you. A single dir /s /a revealed the hidden ones. findstr dug the rest out of any file.

THE LESSON

Hiding and renaming are obscurity, not security. They slow down a curious person for about ten seconds. Against anyone with basic commands, they fail completely. This is one of the most important ideas in all of security.

Debrief 2

Shares Expose Everything

The moment you mapped the Z drive, every file in that folder was within reach. Hidden or not, buried or not, it was all exposed to anyone on the network.

REAL WORLD

Misconfigured network shares are a top cause of real data breaches. Companies accidentally share folders full of passwords, customer data, and backups. Attackers map the drive, run dir /s, and walk away with everything. Exactly what you just did.

Debrief 3

So What Actually Works?

If hiding fails, what protects real data?

  • Encryption: scramble the file so even if found, it is unreadable without a key
  • Access controls: only specific accounts can open the folder, not Everyone
  • Authentication: prove who you are before any access is granted
  • Least privilege: share only what is truly needed, nothing more
THE QUESTION

If you were hiding something important, what would you do differently? That instinct toward encryption is exactly right.
